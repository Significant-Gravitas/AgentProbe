# product-specs Index

## Purpose

Canonical product behavior specifications and the artifacts derived from them.
`platform.md` is the source of truth for product behavior; coverage and current
state documents should stay derivable from it.

## File conventions

- Markdown files describing concrete, testable product behavior.
- Update `platform.md` first when behavior changes.
- Keep implementation details out of this directory unless they are needed to
  make the behavior testable or unambiguous.
- Coverage summaries and checklists must stay in sync with `platform.md`.

## Files

- [README.md](README.md) — How to use and update the product specs
- [platform.md](platform.md) — Canonical product behavior contract
- [current-state.md](current-state.md) — What is currently implemented and covered
- [e2e-checklist.md](e2e-checklist.md) — Coverage map from scenarios to tests
