# Product Specs

This directory defines what AgentProbe should do from the product point of
view.

## Update order

1. Update [platform.md](platform.md) when behavior changes.
2. Refresh [current-state.md](current-state.md) so it describes what is already
   implemented or intentionally deferred.
3. Refresh [e2e-checklist.md](e2e-checklist.md) so coverage expectations stay
   visible and testable.

## Scope rules

- Capture user-visible behavior and operational expectations.
- Keep architecture, toolchain, and enforcement rules in the top-level docs.
- Prefer concrete scenarios over abstract aspirations.
