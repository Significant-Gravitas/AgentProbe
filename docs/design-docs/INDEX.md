# design-docs Index

## Purpose

Durable design decisions and core beliefs that shape how AgentProbe is built.
These docs explain why the repository is structured the way it is and what
principles should survive across refactors and migrations.

## File conventions

- Markdown files only.
- Prefer durable principles and design decisions over sprint-specific notes.
- When a recurring review comment reveals a missing principle, capture it here
  or promote it into code/tooling.
- Do not duplicate detailed implementation plans here; those belong in
  `docs/exec-plans/`.
